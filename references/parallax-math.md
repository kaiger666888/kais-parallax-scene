# 视差计算数学原理

## 核心公式

### 透视投影

摄像机焦距为 `f`，物体在 Z 轴深度为 `d` 时：
- 屏幕位移量：`offset = camera_move × f / (f + d)`
- Z=0（中景）的层完全跟随摄像机移动
- Z→∞（远景）的层几乎不动

### 缩放补偿

远处的层需要放大以维持视觉一致性：
```
scale = 1 + (|Z| / focal_length)
```
- 前景 (Z=-1.5m, f=35mm): scale = 1.043
- 背景 (Z=7.5m, f=35mm): scale = 1.214
- 远景 (Z=20m, f=35mm): scale = 1.571

### 边缘防穿帮

摄像机平移时，平面边缘会暴露。解决方案：
1. **边缘延伸**：UV向外扩展10%，用 content-aware fill 补充
2. **边缘模糊**：蒙版边缘羽化3-5px高斯模糊
3. **运动限制**：限制摄像机X轴移动范围在±3m以内

## 视差量参考表

| 层 | Z(m) | 视差系数(f=35mm) | 3m移动的屏幕位移 |
|----|------|------------------|-----------------|
| 前景 | -1.5 | 0.959 | 0.122m |
| 中景 | 0.0 | 1.000 | 0.000m |
| 背景 | 7.5 | 0.824 | 0.529m |
| 远景 | 20.0 | 0.636 | 1.091m |

视差系数越大，该层随摄像机移动越同步（近景）。
视差系数越小，该层看起来越"远"（背景移动慢）。
